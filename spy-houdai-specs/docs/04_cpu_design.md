# スパイ砲台 CPU砲台 行動パターン設計 v0.1

## 1. 目的

CPU砲台は以下の用途で使用する。

- 人数不足時の補充
- プレイヤー切断時の自動操作
- 1人練習モード
- チュートリアル
- 開発中の自動テスト
- ローカルプロトタイプ検証

本作の中心体験は人間同士のパーティプレイであり、CPUは補助機能として扱う。

## 2. 基本方針

MVPでは、高度なAIやLLMによる会話生成は行わない。  
シンプルなルールベースで実装する。

CPUは以下を行う。

- 行動選択
- 弁明カード選択
- 投票
- 作戦分岐投票
- スパイ時の妨害
- 怪しいコイン使用

## 4. スパイCPU

CPUがスパイになった場合、以下の方針で行動する。

- 序盤は目立たない
- 中盤から妨害する
- 終盤は大胆になる
- 疑惑メーターが高い場合は潜伏を強める
- 怪しいコインは追い詰められた時に使う

## 6. 状況補正

### 自分の疑惑が高い場合

条件：疑惑メーター★★★以上

補正：

- 通常攻撃 +15%
- 偽装攻撃 +10%
- ボス回復 -10%
- ログ撹乱 -5%
- 妨害 -10%

### ボスHPが少ない場合

条件：ボスHP30%以下

補正：

- ボス回復 +15%
- 妨害 +15%
- 通常攻撃 -10%
- 偽装攻撃 -10%

### 拠点耐久が低い場合

条件：拠点耐久30%以下

補正：

- 偽装攻撃 +10%
- ボス回復 -10%
- 露骨な妨害を控える

### 監視対象の場合

補正：

- 通常攻撃 +20%
- 防御 +10%
- 妨害 -15%
- ボス回復 -15%

## 7. 怪しいコイン使用AI

使用候補：

- 自分が投票1位になりそう
- 疑惑メーター★★★以上
- ラウンド3以降
- 最終投票前
- 無実の高疑惑者がいる
- 監視対象を別プレイヤーに移したい

使用確率：

| 状況 | 使用確率 |
|---|---:|
| ラウンド1 | 0% |
| ラウンド2 | 0% |
| ラウンド3 | 10% |
| ラウンド4 | 20% |
| ラウンド5 | 35% |
| 自分が投票1位候補 | +15% |
| 無実の高疑惑者がいる | +10% |

## 8. CPU投票AI

CPUは各プレイヤーに怪しさスコアを計算する。

| 要素 | 加点 |
|---|---:|
| 疑惑メーター | 中 |
| スキャン微弱なノイズ | 中 |
| スキャン強い異常反応 | 高 |
| 監視対象ログの具体ヒント | 高 |
| 直近2ラウンドの異常ログ | 中 |
| 弁明カードとログが矛盾 | +3 |
| 前回投票1位 | +1 |
| 自分を疑ってきた | +1 |

30%の確率でスコア2位以下の候補に投票し、機械的になりすぎないようにする。
