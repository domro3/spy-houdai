# スパイ砲台 ゲームルール詳細

## 1. 行動カード

各プレイヤーは1ラウンドにつき1つの行動を選択する。

## 2. 砲台チーム行動

### 通常攻撃

| 項目 | 値 |
|---|---:|
| ダメージ | 60 |
| 成功率 | 100% |
| 疑惑増加 | なし |

安定した基本攻撃。

### チャージ攻撃

| 項目 | 値 |
|---|---:|
| 成功時ダメージ | 100 |
| 失敗時ダメージ | 30 |
| 成功率 | 75% |
| 暴発率 | 10% |
| 暴発時拠点ダメージ | 5 |

高火力だが失敗や暴発で疑われる可能性がある。

### 防御

| 項目 | 値 |
|---|---:|
| ボス攻撃軽減率 | 50% |
| 妨害軽減率 | 30% |

拠点耐久を守る。

### 修理

| 項目 | 値 |
|---|---:|
| 回復量 | 20 |
| 最大耐久 | 100 |

拠点耐久を回復する。

### スキャン

対象プレイヤー1人を調べる。  
結果は確定情報ではなく、曖昧なヒントとして出す。

結果例：

- 異常反応なし
- 微弱な異常反応あり
- 砲台ログに不自然な欠落あり
- このラウンドの行動に矛盾の可能性あり

## 3. スパイ専用行動

### 偽装攻撃

| 項目 | 値 |
|---|---:|
| 実ダメージ | 30 |
| 表示 | 攻撃ログに混ざる |

攻撃しているふりをする行動。

### ボス回復

| 項目 | 値 |
|---|---:|
| 回復量 | 50 |
| 使用リスク | 高 |

ログ例：

- ボスの装甲が一部再生した
- ボスに謎の回復反応が発生した

### 妨害

対象プレイヤー1人の行動を弱体化させる。

| 対象行動 | 効果 |
|---|---|
| 攻撃 | ダメージ30%減 |
| チャージ攻撃 | 成功率20%低下 |
| 防御 | 軽減率低下 |
| 修理 | 回復量低下 |
| スキャン | 結果が曖昧になる |

### ログ撹乱

そのラウンドのログを一部曖昧にする。

例：

| 通常ログ | 撹乱後ログ |
|---|---|
| ボスに合計280ダメージ | ボスにダメージ |
| 青砲台周辺で異常反応 | どこかで異常反応 |
| スキャン結果：微弱反応 | スキャン結果が不安定 |

## 4. 怪しいコイン

スパイが1ゲームに1回だけ使える救済アイテム。

- ラウンド3以降の投票前に使用
- 成功率40%
- 成功すると2票投票
- 失敗すると疑惑メーター+2
- 使用者は非公開
- 異常ログは全員に表示

## 5. 弁明カード

ラウンド結果後、各プレイヤーは弁明カードを1枚選ぶ。

候補：

- 攻撃しました
- 回復しました
- 防御しました
- スキャンしました
- 妨害されました
- 出力が不安定でした
- ○○が怪しいです
- ○○は怪しくないと思います
- 次は防御が必要です
- スキャン希望です

スパイも嘘の弁明カードを選べる。

## 6. 疑惑メーター

疑惑メーターはスパイ確定情報ではない。  
市民でも高くなることがあり、スパイでも低く保てる。

疑惑上昇要因：

- 投票された
- スキャンで異常反応が出た
- 弁明とログが矛盾
- 妨害発生ラウンドで怪しい位置にいた
- 攻撃宣言が多いのに総ダメージ不足
- 怪しいコイン失敗

疑惑低下要因：

- スキャンで異常なし
- 防御で拠点被害軽減
- 修理で拠点回復
- チャージ攻撃成功
- 他プレイヤーから信頼カード

## 7. 投票

各ラウンド終了後、怪しいプレイヤーに投票する。

- 1人1票
- 追放なし
- 投票1位は次ラウンドで監視対象
- 監視対象は次ラウンドのログが少し多めに公開される

## 8. 中間作戦分岐

ラウンド3終了時、状況に応じて分岐投票を行う。

順調時：

- 通常継続
- オーバードライブ作戦

苦戦時：

- 通常継続
- 緊急弱体化プロトコル

スパイも投票できるため、選択が推理材料になる。

## 9. 自動砲台

切断またはCPU補充時、砲台ロボが操作する。

市民CPU切断引き継ぎ：

| 行動 | 確率 |
|---|---:|
| 通常攻撃 | 70% |
| 防御 | 20% |
| 修理 | 10% |

スパイCPU切断引き継ぎ：

| 行動 | 確率 |
|---|---:|
| 偽装攻撃 | 60% |
| 通常攻撃 | 25% |
| 弱い妨害 | 15% |
