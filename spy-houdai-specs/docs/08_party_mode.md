# Party Mode 仕様 v0.1

## 1. 目的

Party Modeは『スパイ砲台』の標準入口である。

初心者と上級者が混ざっても遊びやすく、ログ読解よりもボスを撃つ爽快感、拠点を守る判断、1戦後の「もう一戦」を重視する。

## 2. Advanced Modeとの役割分担

既存Phase 1の推理寄り要素はAdvanced Modeとして残す。

Party Modeでは、以下を勝敗の中心にしない。

- 複雑なスキャン
- 監視対象ログ
- 最終推理ヒント
- 怪しいコイン
- ログ撹乱
- ラウンド3中間作戦分岐
- スパイ裏勝利
- 詳細な疑惑メーター

## 3. 勝利条件

砲台チーム勝利:

- 5ラウンド以内にボスHPを0にする

スパイ勝利:

- 5ラウンド終了時点でボスHPが残っている
- または拠点耐久が0になる

ゲーム終了後のスパイ当て投票は、勝敗ではなくボーナス・称号・盛り上げ要素として扱う。

## 4. オペレーター指示モデル

Party Modeでは、プレイヤーは砲台そのものではなく、タレットへ作戦信号を送るオペレーターとして扱う。

基本構造:

- オペレーターは自分の端末でコマンドを選ぶ
- 端末からタレットへ作戦信号が送られる
- 戦闘シーンでは、タレットが信号を受けてアクションを実行する
- 公開画面では、オペレーター入力状況、信号の到達、タレット実行結果、公開ログを表示する
- 公開画面では、進行中のスパイ正体、スパイ専用行動名、個別ログは表示しない

この整理により、画面上の主語を以下のように分ける。

| 層 | 役割 | 表示例 |
| --- | --- | --- |
| オペレーター | プレイヤー本人。端末で指示を選ぶ | P1 送信済、P2 待ち |
| 信号 | 選んだ指示がタレットへ届く表現 | 攻撃信号、防御信号、修復信号、ノイズ信号 |
| タレット | 実際にアクションを行う小型砲台 | 発射、バリア展開、修復アーム、通信ノイズ |
| 公開結果 | 全員が見てよい戦闘結果 | ダメージ、修復量、軽減量、異常回復 |

### 戦闘シーンの演出

行動解決時は、各タレットからアクションが行われ、アニメーションと数値ポップで結果を伝える。

| 種別 | 公開演出 | 数値表示 | 秘密情報の扱い |
| --- | --- | --- | --- |
| 撃つ | 青/シアンの信号、砲撃、命中火花 | ボスに `-N` | 通常公開 |
| 守る | 水色の信号、バリア展開、弾き演出 | `軽減 N` | 通常公開 |
| 直す | 緑の信号、修復アーム、拠点復旧 | 拠点に `+N` | 通常公開 |
| 邪魔する | 紫ノイズ、信号乱れ、砲身ブレ | 公開は `ノイズ` 程度 | 誰が邪魔したかは非公開 |
| ボスを助ける | ボス側の異常な回復反応 | ボスに `異常回復 +N` | 原因オペレーターは非公開 |

数値ポップはログの代替ではなく、瞬間的に「何が起きたか」を読むための演出とする。詳細な計算や秘密情報はdebugLogまたは個別ログに留める。

## 5. 砲台チーム行動

Party Modeの砲台チーム行動は3つに絞る。

- 撃つ: ボスにダメージ。迷ったらこれ。
- 守る: ボスの大技や狙い撃ちを軽減する。
- 直す: 拠点耐久を回復する。

直すは通常時18回復を基本とする。
拠点耐久が40以下なら24回復、25以下なら30回復とし、拠点ピンチ時に立て直す理由を強める。
ただし、スパイに邪魔された修理は回復量が下がる。

## 6. スパイ行動

Party Modeのスパイは、味方のふりを自然にできるように通常行動も選べる。

基本行動:

- 撃つ: 砲台チームと同じ通常ダメージ。味方のふりをしたい時に使える。
- 守る: 砲台チームと同じ防御効果。狙い撃ちや大技に自然に対応できる。
- 直す: 砲台チームと同じ修理効果。拠点ピンチ時に自然に振る舞える。

スパイ用行動:

- 弱く撃つ: 撃っているふりをする。通常よりダメージが低い。
- 邪魔する: 誰か1人の行動を少し弱くする。
- ボスを助ける: ボスへ隠し信号を送り、ボスを少し回復する。強いが目立つ。

UIでは、初心者が混乱しないように「撃つ・守る・直す」を基本行動として先に表示し、「弱く撃つ・邪魔する・ボスを助ける」はスパイ用行動として控えめに表示する。

公開画面では「邪魔する」「ボスを助ける」というスパイ専用行動名を進行中に直接表示しない。代わりに「ノイズ」「異常回復」「信号乱れ」など、推理材料になる公開表現へ変換する。

## 7. ボス行動

Party Modeではボス行動を戦略の中心にする。

初期実装のボス行動は以下の4種類。

### normal_attack / 通常攻撃

- 拠点に小ダメージ
- 基本行動
- 拠点が減ると「直す」理由になる

### big_charge / 大技チャージ

- 事前に大技予告を出す
- 拠点に大ダメージ
- 誰かが「守る」を選ぶと大きく軽減
- スパイが守る人を邪魔するとバリアが弱まる
- 未ガード時は特に危険な拠点ダメージになる

### armor_regen / 装甲再生

- ボスが回復しようとする
- 攻撃量が一定以上なら再生阻止
- 攻撃量が足りなければボスHPが少し回復

### target_lock / 狙い撃ち

- 特定プレイヤーを狙う
- 対象者が「守る」を選ぶと大きく軽減
- 対象者が守らない場合、拠点に中から大ダメージ

## 8. ボス定義

ボスはGameCoreにベタ書きせず、データ定義として扱う。

```ts
type BossActionType =
  | "normal_attack"
  | "big_charge"
  | "armor_regen"
  | "target_lock";

type BossDefinition = {
  id: string;
  name: string;
  description: string;
  maxHpByPlayerCount: Record<4 | 5 | 6, number>;
  actionWeights: Partial<Record<BossActionType, number>>;
  specialRules?: string[];
};
```

初期ボス:

- id: `prototype_gigant`
- name: `プロトタイプ・ギガント`
- description: `基本的な攻撃、装甲再生、狙い撃ちを使う標準ボス`

人数別ボスHP:

- 4人: 620
- 5人: 880
- 6人: 1110

行動比率:

- 通常攻撃: 32%
- 大技チャージ: 33%
- 装甲再生: 12%
- 狙い撃ち: 23%

この比率は、拠点耐久0敗北を5人標準で5から10%程度まで増やし、「守る」「直す」を選ぶ理由を作るためのPhase 2暫定値である。

主要ダメージ:

- 通常攻撃: 14 / 守る成功時7 / ノイズあり10
- 大技チャージ: 80 / 守る成功時18 / ノイズあり35
- 狙い撃ち: 56 / 対象者が守る成功時10 / ノイズあり28

### 将来拡張メモ

今後、BossDefinitionにはボスごとの個性を出すために以下を追加できるようにしたい。

- 行動比率
- 固定行動パターン
- ラウンドごとの確定行動
- HP条件で行動変化
- クールダウン
- 連続使用禁止
- 固有行動

例:

- 砲撃型ボス: ラウンド2と4で大技チャージしやすい
- 再生型ボス: HP50%以下で装甲再生が増える
- 暗殺型ボス: 狙い撃ちが多いが、同じ対象を連続で狙わない

## 9. ログ方針

Party ModeのpublicLogは短く、イベント的に表示する。

- 1ラウンド3から4行程度
- 内部計算はdebugLogへ分離
- 数値より「何が起きたか」を優先
- スパイ確定情報は終了まで出さない

### 妨害フィードバック

スパイの「邪魔する」は、全体ログでは「どこかにノイズ」程度に留める。

一方で、妨害された本人には個別ログで明確に伝える。

- 撃つを邪魔された: 砲身がブレた / 攻撃出力が下がった
- 守るを邪魔された: バリアにノイズ / 防御効果が弱まった
- 直すを邪魔された: 修理アーム停止 / 回復量が少なくなった

スパイ本人には「妨害成功」と、どの行動を乱したかを個別ログで表示する。

### シミュレーション集計

ボスHP回復の原因を見分けるため、Party Modeでは以下を分けて集計する。

- `spyBossHelpCount`: スパイが「ボスを助ける」を選んだ回数
- `armorRegenAttemptCount`: ボス行動として装甲再生が発生した回数
- `armorRegenSuccessCount`: 装甲再生が阻止されず、実際にボスHPが回復した回数
- `baseDestroyedRate`: 拠点耐久0で敗北した試合の割合
- `baseDanger40Rate`: 拠点耐久40以下になった試合の割合
- `baseDanger25Rate`: 拠点耐久25以下になった試合の割合
- `averageDefenseCount`: 1試合あたりの守る平均回数
- `averageRepairCount`: 1試合あたりの直す平均回数

## 10. 目標バランス

- 砲台チーム勝率: 55から70%
- スパイ勝率: 30から45%
- 5人標準の拠点耐久0敗北率: 5から10%
- 1ゲーム時間: 5から8分
- 1ラウンドの理解負荷: 低め
- 推理の複雑さ: 軽め
- ボス戦の爽快感: 高め
