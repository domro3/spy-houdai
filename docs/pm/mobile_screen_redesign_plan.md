# Mobile Screen Redesign Plan

作成日: 2026-06-29

ステータス: M5 Closed / Public Alpha v0.1 AI Complete / M6または公開準備へ移行

## 決定

ゲーム画面を大きく変更し、スマホでのプレイを主対象にする。

M4までは、`/board` を共有戦況スクリーン、`/player/:id` を個別作戦端末として分離する方針で検証した。M5では、この情報分離は維持しつつ、Public Alpha v0.1で初見プレイヤーが触る画面をスマホ前提で再設計する。

Public Alpha v0.1では、Board画面を必須にせず、スマホだけで遊べる導線を優先する。Boardは残すが、公開Alphaの主導線ではなく補助表示として扱う。

2026-06-30にM4を終了判定したため、本書の残タスクはM5/Public Alpha v0.1準備として扱う。

## 目的

- 初見プレイヤーがスマホで迷わず1ゲームを進められる
- 次に押す操作が常に分かる
- ボスHP、拠点耐久、ラウンド、危険状態が小さい画面でも読める
- スパイ本人だけが秘密行動を理解できる
- Public Alphaとして雑に見えない最低限の見た目を確保する

## スコープ

### 今回やる

- スマホ縦画面を主対象にしたPlayer画面の再設計
- Boardなしでも戦況確認、行動選択、待機、結果確認まで進められる導線
- 次操作を強く示すステータス/CTAの追加
- 小さい画面で読める戦況サマリー
- 通常行動とスパイ行動の見分け改善
- 送信済み、待機中、解決中、結果表示の状態整理
- Public Alpha用として最低限の見た目を整える

### 今回やらない

- ゲームルールの大幅変更
- 中央Board画面を必須とする体験設計
- オンライン対応
- 本格的なアカウント/マッチング
- 正規版相当のグラフィック制作
- Steam等の販売ページ整備

## 未決事項

| 項目 | 選択肢 | 現時点の扱い |
| --- | --- | --- |
| Board画面を公開Alphaで必須にするか | 必須 / 任意 / スマホ内サマリーに統合 | スマホ内サマリー優先。Boardは任意補助 |
| 1台スマホで複数人が回す導線を残すか | 残す / 隠す | Public Alpha v0.1では隠す。`p1手動 + CPU4基` のソロ導線に絞る |
| Public Alphaの入口 | `/` / `/board` / `/player/p1` / 専用ランディング | `/` をスマホ向け入口にする |
| 横画面対応 | 対応 / 非対応 | Public Alpha v0.1では非対応 |

## 週末公開への影響

この方針変更により、単なるUIブラッシュアップではなく画面再設計になる。週末公開は可能だが、公開範囲は以下に絞る。

- 公開名: Public Alpha v0.1
- 対象: スマホ縦画面で触れるWeb Alpha
- 成功条件: スマホだけで1ゲーム完走、秘密情報漏れなし、次操作が分かる
- 品質基準: 完成版ではなく、遊べる検証版として公開できる

## Go条件

- 390px前後のスマホ幅で主要操作が横にはみ出さない
- 次に押す操作が画面上部または主要CTAとして分かる
- 通常プレイヤーにはスパイ専用行動が出ない
- スマホ内の戦況サマリーから秘密情報が漏れない
- Boardなしでも結果確認まで到達できる
- 1ゲームが最後まで進む
- `npm test`、`npm run build`、`npm run playtest:ai -- --games 100` が通る

## No-Go条件

- スマホで主要ボタンが読めない、押せない
- Player端末だけでは次操作が分からない
- 進行不能バグが残る
- スパイ正体や秘密行動が意図せず見える
- 公開ページでAlpha版であることが伝わらない

次は、入口・結果・再戦導線を含むソロAlpha全体を、初見プレイヤー向けの短い説明文と公開ページ導線としてさらに磨く。

## 入口方針

Public Alpha v0.1では、`/` をスマホ向けの主入口にする。

- 初見プレイヤーは `/` から開始する
- `/board` は共有表示・確認用として残す
- `/player/:id` は内部導線または直接確認用として残す
- 入口画面では、Alpha版であること、スマホ縦画面推奨であること、ゲーム開始方法を短く示す

## 2026-06-30 入口実装メモ

`/` をPublic Alpha v0.1のスマホ入口に変更した。従来の開発Shellは `/dev` へ移動した。

実装内容:

- `/` にPublic Alpha入口を表示する
- `この端末で開始` から、p1を手動操作、残り4基をCPU同期するソロAlphaを開始する
- ソロAlpha中は `SyncedPlayerScreen` と同じHUDを使い、Boardタブを開かなくても戦況、行動、待機、結果へ進める
- `/board`、`/player/:id`、`/debug` は従来どおり残す
- `/dev` に開発用のHost + Player Shellを残す

確認済み:

- 390px幅で `/` 入口に横スクロールなし
- 390px幅で開始後のPlayer HUDにボスHP、拠点耐久、大技チャージ、行動ボタンが表示される
- 390px幅で行動ボタンは75px高を維持
- `撃つ` を押すとCPU補完でROUND 2へ進む
- Board待ち表示にならない
- p1端末にスパイ正体が漏れない
- `npm run typecheck`、`npm test`、`npm run build`、`npm run playtest:ai -- --games 100` が通る

## 2026-06-30 次操作チップメモ

Player端末だけで次に押す操作が分かりにくい問題に対して、下部コマンドカードへ判断チップを追加した。

実装内容:

- 行動選択では、公開ボス予告と拠点耐久から `守る候補`、`直す候補`、`撃つ候補` を表示する
- スパイ本人の端末では `裏回線 / 妨害・偽装・支援` の判断チップを表示する
- スパイ予想、弁明、作戦投票では「候補を1つ押す」系の短い次操作を表示する
- 妨害を受けた直後は `通信ノイズ` チップを表示する

確認済み:

- 390px幅の行動選択で `大技警戒 / 守る候補` が読める
- 390px幅のスパイ予想で `スパイ予想 / 候補を1つ押す` が読める
- 行動ボタンと予想ボタンは44px以上を維持
- 横スクロールなし

## 2026-06-30 結果/再戦導線メモ

Public Alpha v0.1のソロ導線で、結果画面に `もう一戦`、`入口へ戻る`、`Boardで確認` を追加した。

実装内容:

- `SyncedPlayerScreen` の結果カードに任意アクションを差し込めるようにした
- `/` のソロAlphaで、結果フェーズのみ再戦アクションを表示する
- `もう一戦` はスマホでも58px高の主CTAとして表示する
- 再戦時は同じAlpha設定で新規ゲームを開始し、ROUND 1の行動選択へ戻す

確認済み:

- 390px幅で結果画面に横スクロールなし
- 結果画面で勝者、スパイ正体、おまけ投票、称号が読める
- `もう一戦` クリック後にROUND 1へ戻り、結果カードが残らない
- `npm run typecheck`、`npm test`、`npm run build`、`npm run playtest:ai -- --games 100` が通る
