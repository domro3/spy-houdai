# スパイ砲台 アートディレクション

ステータス: Phase 2 Party Mode Alpha向けドラフト。最終アセット仕様ではなく、手動プレイテスト時の画面・演出判断メモとして扱う。

世界観・主要名称の詳細は `docs/design/worldbuilding.md` を参照する。

## 目的

このドキュメントは、ゲーム「スパイ砲台」のグラフィック、UI、演出、配色の基本方針を定義する。

現時点では本番イラストを作り込む段階ではなく、プロトタイプの画面と演出が同じ方向を向くための判断基準を決めることを目的とする。

## 現時点の結論

スパイ砲台のデザイン方針は、以下で進める。

> スパイ砲台の見た目は「読みやすいパーティSF」とする。
> デフォルメされた砲台チーム、怪しいスパイ砲台、巨大な暴走メカボスを使い、青=砲台チーム、紫=スパイ、赤=ボス危険、緑=修理、黄色=警告という意味づけを徹底する。
> ログを読まなくても、攻撃、防御、修理、妨害、ボス支援、大技チャージが一瞬でわかる画面を目指す。
> 特にスパイ妨害は、数字ではなく、通信ノイズ、照準ズレ、黒い火花、短いポップ表示で「邪魔された感」を強く出す。
> 特定ゲームの表現には寄せず、「砲台基地」「通信妨害」「暴走メカ」という質感で独自性を出す。

## デザインコンセプト

### 読みやすいパーティSF

スパイ砲台は、リアル軍事ゲームや暗いサイバーパンクではなく、デフォルメされた砲台、スパイ、巨大ボスが大騒ぎする、見やすいSFパーティゲームを目指す。

画面を見た瞬間に、プレイヤーや観戦者が以下を理解できることを最優先とする。

- 誰が攻撃したか
- 誰が守ったか
- 誰が修理したか
- 誰が妨害されたか
- ボスが何をしようとしているか
- 今、危険なのか有利なのか
- 何が推理材料になりそうか

ログは補助情報として重要だが、ログを読まないと状況がわからない画面にはしない。

## 目指す印象

- 一目で状況がわかる
- 少しコミカル
- ボス戦のワクワク感がある
- スパイ妨害が見ていて悔しい
- 派手だが、ごちゃつきすぎない
- 子どもでも状況を理解できる
- 観戦していても盛り上がる
- プレイヤーの行動結果がすぐ伝わる
- 正体隠匿の怪しさはあるが、暗く重すぎない

## 世界観の方向性

スパイ砲台の世界観は、以下の要素を基本とする。

- 砲台基地
- 管制室
- 暴走メカ
- 通信妨害
- 照準ズレ
- 警告灯
- ボス大技チャージ
- 味方の中に紛れたスパイ
- パイロット脱出と砲台ロボ引き継ぎ
- 作戦ログと疑惑メーター

SFではあるが、リアルな軍事表現には寄せすぎない。重厚で暗い戦争ゲームではなく、「おもちゃっぽいメカが大騒ぎするパーティゲーム」に近い温度感にする。

## 避ける方向性

以下の方向には寄せない。

- リアル軍事
- 暗すぎるサイバーパンク
- ホラー寄りのスパイ表現
- ログを読む前提の情報過多UI
- キャラ絵だけが目立つソーシャルゲーム風
- かっこいいが何が起きたかわからない画面
- 特定ゲームの印象に強く寄る表現

特に以下は避ける。

- インク飛沫を中心にした表現
- 宇宙船クルー風のキャラクター
- 投票・追放ゲーム風のUI
- 既存ゲームに似た丸顔キャラクター比率
- 既存ゲームに似たボタン形状やフォント
- ストリート系の蛍光インク表現
- 魔法エフェクト風の紫表現
- 人狼の村人、占い師、追放を連想させる見た目

## 配色方針

配色は、機能を識別しやすくするために使用する。色そのものをブランド模倣として使うのではなく、ゲーム内の意味を明確に伝えるために使う。

| 要素 | 色の方向性 | 意味 |
| --- | --- | --- |
| 砲台チーム | 青 / シアン | 味方、基本行動、安定 |
| 攻撃 | 青 + 黄色 | 砲撃、命中、火花 |
| 防御 | 水色 / 青 | シールド、軽減、守る |
| 修理 | 緑 / 黄緑 | 回復、復旧、メンテナンス |
| スパイ | 紫 / 黒 / 少量マゼンタ | 妨害、裏回線、ノイズ |
| ボス | 赤 / オレンジ | 危険、敵、怒り、暴走 |
| 警告 | 赤 / 黄色 | 大技、緊急、注意 |
| 成功演出 | 白 / 金 | ナイス行動、盛り上げ |
| 通常背景 | 低彩度のグレー / 生成り / 暗い管制室色 | 情報を支える土台 |

### 推奨カラートークン

実装では、まず以下のような意味ベースのトークンに整理する。

| トークン | 用途 | 色の目安 |
| --- | --- | --- |
| `team-blue` | 味方、通常攻撃、主要ボタン | `#2f80ed` |
| `team-cyan` | 防御、シールド、補助UI | `#35c2d8` |
| `repair-green` | 修理、復旧、成功回復 | `#36b37e` |
| `spy-purple` | スパイ行動、妨害、ノイズ | `#7b3ff2` |
| `spy-black` | 裏回線、異常影、ノイズ輪郭 | `#17121f` |
| `boss-red` | ボス、危険、被害 | `#d83a3a` |
| `warning-yellow` | 警告、火花、注目 | `#f6c945` |
| `success-gold` | 連携成功、称号、結果発表 | `#f7b84b` |
| `panel-bg` | パネル背景 | `#fffefa` |
| `field-bg` | 画面背景 | `#eef3f5` |
| `ink-main` | 本文 | `#1c2027` |

既存CSSの温かい生成り背景はプロトタイプとして使える。ただし、今後は「管制室」「砲台基地」の印象を強めるため、背景は低彩度グレーや淡い青緑を混ぜ、UI上の意味色が目立つようにする。

## 色の重要ルール

### 紫はスパイ専用色として扱う

紫は、スパイ妨害、裏切り、通信異常を示す色として使う。通常攻撃、ボス攻撃、修理、防御には原則として紫を使わない。

スパイの演出は、紫の魔法やインクではなく、以下のような質感で表現する。

- 通信妨害
- グリッチ
- ノイズ
- 照準ズレ
- 黒い火花
- 信号の乱れ
- UIの一時的なバグ

### 赤はボスと危険に使う

赤・オレンジはボス、攻撃予告、大技チャージ、危険状態に使う。常に危険を表す色として扱い、味方側の通常演出には使いすぎない。

### 緑は修理・復旧に使う

緑は回復やメンテナンスの意味を持たせる。医療的な回復というより、機械の復旧・修理として表現する。

### 黄色は警告と火花に使う

黄色は警告、注意、砲撃の火花、成功時の強調に使う。警告時は赤と組み合わせて緊急感を出す。

### 色だけに頼らない

色覚差や小さい画面を考慮し、重要な状態は色だけで判別させない。

- スパイ妨害: 紫 + ノイズ形状 + 「妨害！」表示
- ボス大技: 赤 + 警告アイコン + チャージゲージ
- 修理: 緑 + レンチ + 火花が消える動き
- 防御: 水色 + 盾 + 弾く動き
- 攻撃: 青/黄色 + 砲弾 + 命中火花

## キャラクター方針

### 砲台チーム

砲台チームは、人間キャラクターではなく、デフォルメされた小型ロボット砲台として表現する。

基本方針:

- 青 / シアン系
- 丸みのあるメカ
- 表情より動きで感情を出す
- 役割ごとに形を少し変える
- 攻撃、防御、修理の状態が見た目でわかる
- プレイヤー色はアクセントとして使い、役割色と混同しない

役割別の方向性:

| 役割 | 見た目の方向性 |
| --- | --- |
| 攻撃砲台 | 大砲、弾倉、照準、アンテナ |
| 防御砲台 | 盾、厚い装甲、バリア発生装置 |
| 修理砲台 | レンチ、ライト、工具アーム、修理ドローン |
| 自動砲台ロボ | 操縦席が空、オートランプ、簡易AI表示 |

### オペレーターと信号

プレイヤーはタレットそのものではなく、作戦端末からタレットへ信号を送るオペレーターとして扱う。

画面では、以下の3段階が自然につながって見えるようにする。

1. オペレーターが端末でコマンドを選ぶ
2. 端末からタレットへ作戦信号が飛ぶ
3. タレットがアクションを実行し、数値ポップで結果が出る

オペレーターは人物絵を大きく見せるより、端末UI、ヘッドセット風アイコン、P1/P2などの信号チャンネルとして表現する。主役はあくまでアリーナ上のタレットとボスに置く。

信号の見せ方:

| 信号 | 色・質感 | タレット実行 | 数値ポップ |
| --- | --- | --- | --- |
| 攻撃信号 | 青/シアン + 黄色火花 | 砲撃、命中火花 | ボスに `-N` |
| 防御信号 | 水色 + 盾形状 | バリア展開、弾き | `軽減 N` |
| 修復信号 | 緑 + レンチ/修復粒子 | 修復アーム、拠点復旧 | 拠点に `+N` |
| 妨害信号 | 紫 + 黒いノイズ | 砲身ブレ、信号乱れ | 公開は `ノイズ` |
| ボス修復信号 | 紫/赤 + 異常脈動 | ボスコア回復反応 | ボスに `異常回復 +N` |

妨害信号とボス修復信号は、進行中の公開画面で発信者を特定できる見せ方にしない。紫の信号は「怪しい通信異常」として見せ、スパイ正体は終了まで隠す。

### スパイ

スパイは、味方の中に紛れたイタズラ担当として表現する。完全な悪役ではなく、「怪しいが少し愛嬌がある」存在にする。

基本方針:

- 紫 / 黒 / 少量マゼンタ
- 小型ロボットまたは怪しい砲台
- フード、仮面、ノイズ顔、怪しい目などの要素
- 正体がバレるまでは味方に見える余地を残す
- 妨害時にだけ紫ノイズやグリッチが強く出る

避ける表現:

- ホラーすぎる見た目
- 殺人鬼のような表現
- 既存の宇宙人・クルー風デザイン
- 魔法使い風の紫エフェクト
- 正体公開前からスパイだと断定できるシルエット

### ボス

ボスは、巨大な暴走メカとして表現する。怖さは必要だが、リアルな怪物ではなく、少しデフォルメされた巨大兵器にする。

基本方針:

- 赤 / オレンジ系
- 巨大な砲台、暴走兵器、機械生命体のような印象
- 大きなコア、赤い目、大砲、警告灯
- 大技チャージ時に色や形が強調される
- 怖いがコミカルさも少し残す

`プロトタイプ・ギガント` は、最初の標準ボスとして「試作機が暴走した巨大砲台」という方向で進める。

## UI方針

UIは「かっこいい」より「読める」を優先する。画面が派手でも、現在の判断に必要な情報が埋もれてはいけない。

基本ルール:

- ボタンは大きくする
- アイコンを大きくする
- 文字は短くする
- 状態表示は色と形で判別できるようにする
- 行動結果は画面中央寄りに出す
- ログは補助扱いにする
- プレイヤーが次に何をすべきか分かるようにする
- 中央画面では秘密情報を出さない
- プレイヤー操作欄では自分の役職と行動可否を明確に出す

### 画面内の優先順位

中央画面の視線誘導は、以下の順にする。

1. ボスの現在状態
2. ボスの次の行動予告
3. ボスHP / 拠点耐久
4. ラウンドとフェーズ
5. プレイヤーの入力状況
6. 疑惑メーター / 監視対象 / 推理ヒント
7. 公開ログ

プレイヤー操作欄の視線誘導は、以下の順にする。

1. 自分の役職
2. 今選ぶべき行動
3. 行動ボタン
4. 対象選択
5. 自分にだけ見えるログ
6. 補助説明

## 行動ボタン方針

行動ボタンは、通常行動とスパイ行動を明確に分ける。

通常行動:

- 撃つ
- 守る
- 直す

スパイ行動:

- 弱く撃つ
- 邪魔する
- ボスを助ける

Advanced Modeでは以下も扱う。

- チャージ攻撃
- スキャン
- 偽装攻撃
- ログ撹乱

通常行動は青・緑系を中心にする。スパイ行動は紫・黒系を中心にする。

行動名は短く、Party Modeでは既存の `撃つ`、`守る`、`直す`、`弱く撃つ`、`邪魔する`、`ボスを助ける` を優先する。Advanced Modeでは詳細な行動名を使ってもよい。

## 行動演出方針

### 撃つ

攻撃したことが一瞬で伝わるようにする。

演出例:

- 弾が飛ぶ
- 命中時に小爆発
- 黄色い火花
- ダメージ数字
- 連携時は少し派手にする

### 守る

防御したこと、防御が役に立ったことが伝わるようにする。

演出例:

- 盾アイコン
- 水色のバリア
- 攻撃を受けた時に弾く演出
- ダメージ軽減表示
- 守れた時の気持ちよい反応

### 直す

修理により立て直していることが伝わるようにする。

演出例:

- レンチアイコン
- 緑のプラス表示
- 火花が消える
- 砲台のヒビや煙が減る
- 修理完了時に明るく光る

### 邪魔する

スパイ妨害は、スパイ砲台の見た目の中で最重要演出とする。数字だけでなく、見た目で「邪魔された」と感じられるようにする。

演出例:

- 紫ノイズ
- 照準ズレ
- 画面ブレ
- 黒い火花
- UIの一時的なバグ
- 「妨害！」表示
- 攻撃演出が弱くなる
- 砲台の動きがぎこちなくなる

スパイ妨害は、プレイヤーが見て少し悔しくなるくらい分かりやすくする。ただし、スパイ本人が即座に確定しないよう、中央画面では「どこかに異常が起きた」見え方を残す。

### ボスを助ける

ボスを助ける行動は、スパイ側の重要行動として扱う。ただし、露骨すぎるとすぐにスパイがバレるため、演出は「怪しいが断定しにくい」程度に調整する。

演出例:

- ボスに紫のエネルギーが少し流れる
- ボスのチャージがわずかに進む
- ボスのHPが少し回復する
- UIに一瞬だけノイズが出る
- 観察している人には違和感が残る

### スキャン

Advanced Modeのスキャンは、推理材料を得たことがわかるようにする。

演出例:

- レーダー波
- 対象カードの一時ハイライト
- 微弱ノイズ / 強い反応 / クリアの3段階表示
- 正解を断定しない分析画面

### ログ撹乱

Advanced Modeのログ撹乱は、情報が壊された不安が伝わるようにする。

演出例:

- ログの一部にノイズ帯
- 文字化け風の短い乱れ
- 行間に「通信品質低下」表示
- 疑惑メーターの周辺に小さなノイズ

## ボス大技チャージ演出

ボスの大技チャージは、ゲーム画面の主役として扱う。プレイヤーが「次は守った方がよさそう」と直感できるようにする。

演出例:

- 赤い警告UI
- チャージゲージ
- 警告灯
- サイレン風の表示
- 狙われているプレイヤーの強調
- ボスのコア発光
- 大技発動直前の画面揺れ

ボス大技は、守る・修理する理由を作る重要演出とする。

## 情報表現のルール

### 公開情報と秘密情報を分ける

中央画面は全員が見るため、スパイの正体や各プレイヤーの正確な行動を断定表示しない。公開ログは推理材料を出す場所として扱う。

プレイヤー操作欄では、自分の役職、自分が選べる行動、自分だけに見える結果を明確にする。

### ログは短く、画面演出で補う

ログ文は1行で読める長さにする。行動結果の主役はログではなく、画面上の反応と短いポップ表示にする。

良い例:

- `青砲台周辺で微弱な異常反応`
- `防御フィールドが大技を軽減`
- `通信ノイズにより火力が低下`

避ける例:

- 長い説明文
- ルール説明を毎回表示するログ
- 誰がスパイかを早すぎる段階で断定するログ

## レイアウト方針

### MVPの1PC画面

MVPでは、1つのPC画面内に中央画面と複数プレイヤー操作欄を並べてよい。

推奨構成:

- 上部: ゲーム開始、人数、モード、Seedなどのセットアップ
- 左側: 中央状況、ボスHP、拠点耐久、ボス予告
- 右側: 現在フェーズの操作
- 下部: プレイヤーカード、公開ログ、デバッグログ

### 将来の中央画面

中央画面は「観戦しても盛り上がる共有画面」にする。

優先表示:

- ボスの見た目とアクション予告
- HP / 拠点耐久
- ラウンド進行
- 行動結果の大きな演出
- 疑惑メーター
- 短い公開ログ

### 将来のプレイヤー画面

プレイヤー画面は「秘密を確認して素早く選ぶ画面」にする。

優先表示:

- 自分の役職
- 今の行動選択
- 対象選択
- 弁明カード
- 投票
- 個人ログ

## アイコン方針

現時点では `lucide-react` のアイコンを仮アイコンとして使う。将来の本番アイコンでは、以下の意味が一目でわかることを優先する。

| 行動 | アイコン方向 |
| --- | --- |
| 撃つ | 砲弾、照準、火花 |
| 守る | 盾、バリア |
| 直す | レンチ、工具、修理アーム |
| スキャン | レーダー、虫眼鏡、波形 |
| 邪魔する | 断線ケーブル、ノイズ、ズレた照準 |
| ボスを助ける | ボスコア、裏回線、紫信号 |
| 投票 | 投票札、照準ロック、警戒マーク |
| CPU/砲台ロボ | 小型ロボ、オートランプ |

アイコンは装飾ではなく、短いボタン文言を補助するために使う。

## アニメーション方針

複雑なアニメーションは後回しでよい。最初はCSSトランジションと短い状態変化で、以下を優先する。

1. ボスHPが減る / 回復する
2. 拠点耐久が減る / 修理される
3. スパイ妨害のノイズが入る
4. ボス大技チャージが高まる
5. 防御が攻撃を軽減する
6. 結果発表で称号が出る

アニメーションは0.15秒から0.5秒程度を基本とし、入力や状態確認を邪魔しない。

まずは、ゲームとして何が起きたか伝わることを優先する。

## 既存ゲームとの差別化方針

配色は、青=味方、赤=敵、緑=回復、黄色=警告という一般的なゲームUIの機能色をベースにする。

ただし、特定作品の印象に寄らないように、以下を守る。

- インク表現に寄せない
- 宇宙船クルー風にしない
- 投票・追放UIにしない
- 既存ゲーム風のキャラ比率にしない
- 既存ゲーム風のボタン形状にしない
- 特定作品のフォントやロゴ感に寄せない
- 派手さより「砲台基地」「通信妨害」「暴走メカ」の質感を優先する

## スパイ砲台らしさ

スパイ砲台らしい見た目は、以下の要素で作る。

- 砲台基地
- 暴走メカボス
- 通信妨害
- 照準ズレ
- 紫ノイズ
- 警告灯
- 大技チャージ
- 修理と防御の必要性
- 味方の中に紛れたスパイ
- 砲台ロボの自動制御
- 怪しいコインや作戦分岐による揺さぶり

特に、スパイの演出は「魔法」や「インク」ではなく、「通信妨害」「ノイズ」「誤作動」として表現する。
