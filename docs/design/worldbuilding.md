## 基本コンセプト

「スパイ砲台」は、プレイヤーが人間ではなく、コアを守る自動防衛AIシステムになりきって遊ぶゲーム。

プレイヤーはそれぞれ、コアを防衛するAIユニットとして行動する。人間がAIになりきって、作戦入力、砲台制御、防衛判断を行うイメージ。

ゲーム上のプレイヤー名は以下の形式を想定する。

- `コアガードAI-01`
- `コアガードAI-02`
- `コアガードAI-03`

プレイヤー側システム名は、現時点では以下を採用候補とする。

- コアガード・システム

## 守る対象

守る対象はリンクコア。

リンクコアは単なる電波通信装置ではない。世界全体の通信、AI同士の同期、人間の意思や判断信号までをつなぐ中枢である。

正式名称の候補として、以下を置く。

- ニューロン・リンクコア

ゲーム中では短くリンクコアと呼ぶ。

### リンクコアの役割

リンクコアは、世界中の「つながり」を支える中枢コア。

ここでいう通信は、電波やデータ通信だけではなく、以下を含む。

- AI同士の同期
- 防衛システムの作戦信号
- 人間からAIへの意思信号
- 判断、命令、願いのような抽象的な情報
- 世界全体をつなぐネットワークの基盤

リンクコアが破壊されると、世界の通信と意思のつながりが断たれる。

## プレイヤー側

プレイヤーは、リンクコアを守るために作られた自動防衛AI。

名称:

| 要素 | 名称 |
| --- | --- |
| 個体名 | コアガードAI-01 など |
| システム名 | コアガード・システム |

### コアガードAIの役割

コアガードAIは、リンクコアを守るために砲台を制御する。各AIは作戦を入力し、敵の襲来に対応する。

ゲーム上はプレイヤーがそれぞれのAIになりきって行動する。

### 表現イメージ

- 人間が直接戦うのではない。
- プレイヤーはAIユニットとして防衛システムを動かす。
- 見た目が似ている理由は、同じ規格の防衛AIユニットだから。
- 役職や状態は各AI端末の内部状態として扱う。

## 敵側

敵勢力名はアンリンク。

リンクコアが世界をつなぐ存在であるのに対して、アンリンクはそのリンクを切断、分断、破壊しようとする存在。

### アンリンクの基本設定

アンリンクは、リンクコアの接続網を壊す敵性存在。世界の通信、AIの同期、人間の意思信号を切り離そうとする。

通常の敵やボス個体は、生まれた時から敵側として存在している。元味方ではない。

## ボス個体の命名

通常ボス個体は、以下の形式で命名する。

- アンリンクMk-01
- アンリンクMk-02
- アンリンクMk-03

`Mk` は `Mark` の略で、「型」「モデル」「世代」を表す。

### ボス命名例

| 名称 | 位置づけ |
| --- | --- |
| アンリンクMk-01 | 最初に確認された基本型アンリンクボス |
| アンリンクMk-02 | 派生型、改良型、別能力型 |
| アンリンクMk-03 | さらに強力な個体 |
| アンリンクMk-X | 正体不明、特殊個体の候補 |

現時点では、最初のボス個体としてアンリンクMk-01を使う候補とする。

## スパイの設定

スパイは、単なる裏切り者ではない。また、生まれた時から敵側の存在でもない。

スパイは、コアガードAIのうち1体がバグによって敵側に同期しかけている状態。

候補名称:

- バグリンク
- バグ同期

現時点では、世界観用語としてバグリンクを優先候補にする。

### スパイが味方と同じ見た目である理由

スパイは元々コアガードAIであり、同じ防衛システムに属している。そのため、見た目や基本仕様は他のプレイヤーと同じ。

しかし内部では、バグによってアンリンク側の信号に引っ張られている。

### スパイの物語上の位置づけ

スパイは完全な敵ではなく、敵になりかけている味方。最終的には、仲間が異変に気づき、バグをクリアすることで正気に戻る。

ストーリーとしては、スパイを倒す、処刑する話ではなく、「異変に気づいて、ラストボスのようになる前に救う」方向にする。

## ラストボスの設定

ラストボスは、生まれた時から敵だったアンリンクとは異なる。

ラストボスは、かつては味方だった旧型の防衛AI。昔はリンクコアを守る存在だったが、バグによって敵側になってしまった。

候補名称:

- ロストガードAI

### ロストガードAIの基本設定

ロストガードAIは、かつて最初期に作られたコアガードAI、または旧防衛AI。リンクコアを守るために作られたが、バグによって判断が歪んだ。

本来は世界を守ろうとしていた。しかし、バグによって以下のように誤認するようになった。

- つながりは危険である。
- 通信は争いを生む。
- 世界を守るには、すべてのリンクを切断すべきである。
- リンクコアを停止させることが最善である。

その結果、リンクコアを守る側から、リンクコアを破壊しようとする側になった。

## スパイとラストボスの関係

スパイは、ロストガードAIと同じバグに感染しかけている存在。

ただし、スパイはまだ完全には敵になっていない。仲間が異変に気づき、バグをクリアできれば正気に戻る。

この構造により、物語のテーマは以下になる。

> 壊れかけた仲間を、敵として倒すのではなく、異変に気づいて救う。

スパイはゲーム上は敵陣営として動くが、物語上は「救える存在」として扱う。

## 物語の基本あらすじ

世界の通信と意思をつなぐ中枢、リンクコア。そのコアを守るため、自動防衛システム「コアガード・システム」が稼働していた。

プレイヤーは、コアガードAIの1体として起動する。砲台を制御し、リンクコアへ迫る敵性存在「アンリンク」を迎撃する。

しかし、防衛システムの中に異常が発生する。仲間のコアガードAIの1体が、アンリンク側の信号に引っ張られ始めていた。

それは、かつて旧防衛AIをロストガードAIへ変えてしまったものと同じバグだった。

リンクコアを守れ。アンリンクを撃退せよ。そして、仲間が完全に壊れてしまう前に、その異変に気づけ。
